#Manual Testing Web Project

This repository contains a manual testing project created to demonstrate my QA manual testing skills on a web application.

##Aplication Under Test
- Type: Web application (E-commerce demo)
- Module tested so far: Login, Registration
- Environment: Public demo environment

##Scope
- Manual testing of Login and Registration functionality
- Input validation and boundary testing
- Usability and security-related scenarios
- Anti-bot and multiple login attemps behavior

##Test Types Covered
- Functional Testing
- Negative Testing
- Boundary Value Analysis
- Usability Testing
- Exploratory Testing
- Basic Security Testing

##Test Artifacts
- Test Cases(Excel)
- Login and Registration test cases covering valid scenarios, invalid inputs and edge scenarios(whitespace, length limits, duplicate data)
- Test cases were executed and updated with Actual Results and Pass/Fail status

##Bug Reports  
-Failed cases were analyzed and documented as Bug Reports
-Bugs were clasified by type and severity
-Identified issues include:
      -Missing protection for multiple failed login attemps(security)
      -Anti-bot delay and unclear messaging
      -Usability and validation-related issues

##Tools Used
- Browser: Google Chrome
- Documentation: Microsoft Excel
- Version Control: Git & GitHub

##Curent Status
- Test cases for Login and Registration modules created and executed
- Actual Results and Pass/Fail status updated
- Bug Reports created for failed test cases
- Repository structured to reflect a complete manual testing process
